### DataSource
- 앱이 사용하는 원천 데이터
  - 파일, 텍스트, JSON, DB, CSV, RDBMS, NoSQL 등
- 대부분의 앱은 데이터를 받아와 화면에 보여준다.

#### 역할
- 외부 데이터 저장소와 통신
- CRUD 작업 수행
- 데이터를 가져오고 정리, 보여주기 좋게 만드는 역할을 한다.


### DataSource 구현 방식 정리
- 상황에 따라 이름 짓기
  - 저장소 위치 기준 -> 어디에 저장?
  - 기술 스택 기준 -> 어떤 기술로 구현?
- 클래스 이름을 통해 용도와 구현 방식을 바로 알 수 있도록

# 정리
- DataSource 는 앱이 데이터를 가져오는 출발점이다.
- 다양한 형태(JSON, CSV, DB 등)가 존재
- 데이터를 받아와서 객체로 변환하면 앱에서 쉽게 사용 가능

### readLines & readText
- readText() : 파일의 처음부터 끝까지 이어진 텍스트 덩어리 (반환: String)
- readLines() : 각 줄이 요소가 된 리스트(반환: List<String>)
