### 제네릭

- 사용시점에 타입을 원하는 형태로 정의한다.
- 타입 안전(type safe) 효과

```kotlin
List<E> class
Map<K, V>
class 
```

#### 타입이 없을 때의 문제점
- 런타임 에러가 나기 쉽다.
- IDE가 컴파일 에러를 미리 찾을 수 없다.

### 열거형(ENUM)
- 정해둔 값만 넣어둘 수 있는 타입
- when 조합을 통해 모든 처리를 강제할 수 있다.
- enum 은 값 타입이라 동등성 검사를 안해도 된다. 
- 간단하게 쓰자~
```kotlin
enum class AuthState {
    AUTHENTICATED, UNAUTHENTICATED, UNKNOWN
}

fun something(authState: AuthState) {
    when (authState) {
        AuthState.AUTHENTICATED -> println()
        AuthState.UNAUTHENTICATED -> println()
        AuthState.UNKNOWN -> println()
    }
}
```


### 문자열 조작
- ${수식} 을 활용한 문자열 결합이 가능하다.
- `println("Your name is ${name.length} characters long")`

#### 문자열 처리
- `substring` : 일부 떼어내기
- `replace` : 일부 치환
- `split` : 분리
- `lowercase()`,`uppercase()` : 대소문자 변경
- indexOf() : 문자열 위치 찾기

### String pool
- String은 String pool 이 존재한다.
- 글자를 사용할 때는 이 String pool 을 먼저 보게 된다.
- 런타임 시점에 생성되는 값들은 String pool에 생성되지 않고 일반 Heap 메모리에 생성
