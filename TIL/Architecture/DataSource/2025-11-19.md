# DataSource

> 앱이 사용하는 원천 데이터  
데이터를 수신 및 처리하는 역할

- DataSource의 역할
    - 외부 데이터 저장소와 통신
    - Raw 데이터 수신 및 처리
    - CRUD(Create, Read, Update, Delete) 작업 수행


- 데이터와 뭐가 다를까??
    - DataSource는 데이터를 주는 애 (데이터 수신 및 처리)


- DataSource의 종류
    - Text (.txt 등)
    - File (로컬 파일)
    - JSON (JavaScript Object Notation, 웹 API에서 자주 사용)
    - XML
    - CSV (Comma-Separated Values, 엑셀같은 형식)
    - RDBMS (MySQL, PostgreSQL… 관계형 DB)
    - NoSQL (MongoDB, Firebase Firestore… 등)


- DataSource 상황에 따라 이름 짓기 — ai한테 부탁하자~
    - 저장소 유형별 구현체 (Local-, Remote-, Cached-, File-, Database-, Network-)
    - 구체적인 기술 스택별 구현체 (RoomDB-, Sqlite-, …)


- CSV
    - CSV는 JSON 대비 Key가 없고 내용만 있기 때문에 Json 대비 절반의 메모리만 사용
    - csv 맨 윗줄은 제목줄
