- 동기(Synchronous) vs 비동기(Asynchronous)

    ### 🔹 동기(Synchronous)
  - 코드가 **순서대로 차례대로 실행**
  - 한 작업이 끝날 때까지 **프로그램이 대기**
  - 실행 순서가 예측 가능

  ### 🔹 비동기(Asynchronous)
  - 여러 작업이 실제 병렬은 아니지만 **대기하지 않고 다음 작업을 시작**
  - 작업 완료 순서를 예측하기 어려움
  - **I/O, 네트워크 요청 같은 오래 걸리는 작업**에 적합

---

- 동시성(Concurrency) vs 병렬성(Parallelism)

| 개념 | 설명 |
|------|------|
| **동시성(Concurrency)** | 여러 작업을 번갈아 실행하여 **동시에 실행되는 것처럼 보이는** 동작 |
| **병렬성(Parallelism)** | 여러 작업을 **실제로 동시에** 처리하는 것 |

- 비동기 작업은 **동시성과 병렬성 모두를 포괄하는 개념**

- 스레드(Thread)
  - 가능하면 스레드를 직접 다루기보다 더 안전한 방식 사용을 고려
  - 스레드는 디버깅, 테스트가 어려움
  - 운영체제에서 무겁게 관리되는 리소스(Heavy weight)

---

- 코루틴(Coroutine)
  - **경량(lightweight)** 스레드
  - 일반적으로 하나의 스레드에서 실행되지만 언제든 **일시 중단(suspend)** 가능
  - **코루틴은 스코프에 의존적**
  - `join()` → 코루틴이 끝날 때까지 **대기**

---

- Dispatcher 
  - 코루틴이 **어떤 스레드에서 실행될지** 지정하는 요소

  ### 🔹 `Dispatchers.Default`
    - CPU 연산에 적합
    - 코어 수에 비례해 스레드를 생성
    - 무거운 계산 작업에 사용

    ### 🔹 `Dispatchers.Main`
  - **메인(UI) 스레드**
  - UI 업데이트 시 사용

  ### 🔹 `Dispatchers.IO`
  - 파일, 네트워크, DB 등 **I/O 작업에 최적화**
  - 많은 I/O 작업을 효율적으로 처리하도록 설계
