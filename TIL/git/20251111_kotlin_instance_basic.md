# Kotlin

## 오늘 배운것

객체지향 설계 리뷰
- Bottom up 방식이 Top down 방식보다 쉽다.
- Top down방식은 중간에 수정이 많이 일어난다.
- Bottom up 방식은 특성들이 공통적으로 있는것들을 묶어서 인터페이스들을 달아준다.
- 객체지향의 꽃은 인터페이스이다. 인터페이스를 많이 달아도 상관없다.
- 설계는 인터페이스 다는것을 접근하다가, 속성이 필요하다면 추상클래스로 변경한다.(추상 클래스는 다중 상속이 불가능하기때문에)

개념
- 동등비교
  - 자바에서 equals는 실제 주소비교,
    코틀린에서는 ==
  - List에서는 동일성 판단을 equals로 판단
    Set에서는 동일성 판단을 해시코드로 판단
  - 해시코드는 이미 계산된 값
  - 동일한 객체는 같은 해시값을 가진다.
    하지만 같은 해시값이라도 동일한 객체는 아니다.

- 복사
  - data class 의 copy()는 왜 깊은복사가 아니고 얕은복사를 제공하는가? -> 복사했는데 안변할거라는것이 default
  - 메모리와 cpu를 더 많이 소모하고, 순환참조 문제가 생길수 있기 때문
  - 깊은 복사가 필요한 경우, 개발자가 구현하면 됨
