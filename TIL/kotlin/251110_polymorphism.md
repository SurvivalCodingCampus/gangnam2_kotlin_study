# Polymorphism 다형성

다형성의 객체는 **런타임 시에는 생성한 인스턴스로 결정**된다.  
컴파일 시에는 선언한 타입으로 본다.

## 타입 캐스팅 `as` keyword
타입 캐스팅할 때 `as` 키워드를 사용해서 할 수 있다.

## 타입 체크 `is` keyword
타입 캐스팅은 예외 발생이 있을 수 있기 때문에 캐스팅 전 `is`로 타입이 일치한지 확인해야 한다.

## 정리

### 타입간의 대입
- 상속에 의한 **is-a 관계**가 성립한다면, 인스턴스를 슈퍼 클래스 타입의 변수에 대입할 수 있다

### 인스턴스를 담는 상자의 타입과 실제 내용의 타입의 역할
- 어떤 멤버를 이용할 수 있는가는 상자의 타입이 결정한다.
- 멤버가 어떻게 움직이는지는 내용의 타입이 결정한다

### 다형성
- 같은 부모를 가지는 다른 인스턴스를 동일시하여, 슈퍼 클래스 타입에 담을 수 있다
- 마찬가지로, 부모 클래스 타입의 인수나 리턴 값을 이용하여, 다른 클래스를 모아서 처리 가능
- 동일시 취급 해도, 각각의 인스턴스는 각 클래스의 정의를 따르고 다른 동작을 한다.
