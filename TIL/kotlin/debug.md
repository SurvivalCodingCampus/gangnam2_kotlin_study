# 디버깅

## 디버깅

디버깅은 소프트웨어의 오류를 식별하고 수정하는 과정입니다

## 디버깅의 중요성

디버깅은 소프트웨어가 올바르게 작동하는지 확인하는 데 필수적입니다. 디버깅하지 않으면 소프트웨어가 오류가 발생하거나 제대로 작동하지 않을 수 있습니다.

## 디버깅의 기술

- 로깅
    - 로깅은 코드가 실행되는 동안 발생하는 이벤트를 기록하는 데 사용할 수 있습니다.
- 브레이크포인트
    - 브레이크포인트는 코드의 특정 지점에서 실행을 중지하는 데 사용할 수 있습니다.
- 디버거
    - 디버거는 디버깅을 도와주는 도구입니다. 다양한 기능을 제공하며 디버깅을 더 쉽게 만들 수 있습니다.
- 스택 추적
    - 스택 추적은 호출 스택을 추적하여 코드가 실행 중인 위치를 확인하는 데 사용할 수 있습니다.

### 로깅

print() 함수를 활용하는 방법.

### 브레이크포인트

Debug 모드로 실행하여 원하는 위치에서 코드를 멈출 수 있다

### 디버거

디버그 모드로 실행하여 브레이크 포인트에서 멈추거나 에러가 나면 다양한 도구를 활용하여 에러를 찾는데 도움이 된다

### 스택 추적

어떤 경로로 코드가 실행되었는지 추적할 수 있다

## IntelliJ 디버깅 단축키 (macOS 기준)

### 디버깅 실행 및 중단 관련

- `⌃ (Control) + D` (Debug): 현재 구성을 디버그 모드로 실행합니다.
- `⌘ (Command) + F8` (Toggle Breakpoint): 현재 커서 위치에 브레이크포인트를 설정하거나 해제합니다.
- `F9` (Resume Program): 다음 브레이크포인트가 나올 때까지 실행을 재개합니다.
- `⌥ (Option) + F10` (Show Execution Point): 현재 디버깅이 멈춰있는 코드 위치로 에디터 화면을 이동시킵니다.

### 코드 단계별 실행 (Stepping) 관련

- `F8` (Step Over): 현재 줄을 실행하고 다음 줄로 이동합니다. 메서드 내부로는 들어가지 않습니다.
- `F7` (Step Into): 현재 줄에 메서드 호출이 있을 경우, 해당 메서드 내부로 진입합니다.
- `⇧ (Shift) + F8` (Step Out): 현재 메서드의 나머지 실행을 마친 후, 호출한 상위 메서드의 다음 줄로 복귀합니다.
- `⌥ (Option) + F9` (Run to Cursor): 커서가 위치한 줄까지 코드를 실행하고 멈춥니다.

### 값 확인 관련

- `⌥ (Option) + F8` (Evaluate Expression): 실행이 멈춘 상태에서 변수나 복잡한 표현식의 값을 즉시 평가하여 확인하는 창을 엽니다.
