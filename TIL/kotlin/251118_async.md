# 동기 vs 비동기
- 작업 완료를 기다리는 방식의 차이
- **동기**: 한 작업이 끝날 때까지 기다렸다가 다음 작업을 시작
- **비동기**: 작업을 실행한 후 결과를 기다리지 않고 다른 작업을 시작

# Blocking vs Non-blocking
- **Blocking**: 작업을 실행할 때 현재 스레드를 멈추고 작업이 끝날 때까지 기다림
- **Non-blocking**: 작업을 실행할 때 현재 스레드를 멈추지 않고 다른 작업을 계속 진행하는 방식

동기와 비동기는 작업 순서와 실행 방식을 말하는 개념, blocking과 non-blocking은 작업을 실행하는 방식에서 스레드가 다른 작업을 차단하는지 여부를 다룬다.

# 동시성 vs 병렬성
- 여러 작업을 처리하는 방식에 대한 차이
- **동시성**: 여러 작업을 번갈아가면서 처리
- **병렬성**: 여러 작업을 실제로 동시에 처리

## 정리
- 하나의 교사가 여러가지 일을 하는 것 (싱글 코어) -> **동시성**
- 교사와 조교가 함께 일을 하는 것 (멀티 코어) -> **병렬성**
- 순서대로 실행하는 것 -> 동기
- 동시에 실행하는 것 -> 비동기 (동시성, 병렬성을 모두 표현할 수 있음)

# 코루틴
- 비동기(async) 프로그래밍 방식으로 코루틴(Coroutine)이 있다.
- **코루틴 기본 동작은 순차 실행이다.**
- 코루틴은 스코프에 의존적이다.
- 새로운 코루틴을 시작하는 함수를 **코루틴 빌더**라고 한다.

## `suspend` 함수
- 코루틴을 정의할 때 사용하는 키워드
- 비동기 작업을 일시 정지하고 다시 시작할 수 있게 하는 역할

## 코루틴 빌더 종류
- launch
- async / await
- runBlocking

등이 존재한다.

## 코루틴 스코프 정리
- 모든 코루틴을 추적하여 실행되어야 하는 시기를 관리하는 구성 요소
- 모든 비동기 작업은 특정 스코프에서 실행되어야 한다
