# 문자열 조작

### 문자열 결합

```kotlin
println("Hello" + "Kotlin")
val name = "lee"
println("Hello, $name")
println("${name.length} characters long")

```

1. 문자열 템플릿 (String Template)  
   $ 기호를 사용하여 변수를 직접 문자열에 삽입합니다.

2. `+` 연산자  
   가장 기본적인 방법으로 문자열을 결합합니다.

3. plus() 메서드  
   Java의 concat()와 유사한 메서드입니다.

```kotlin
val str1 = "hello"
val str2 = "kotlin"
val result = str1.plus(str2)  // "hellokotlin"

```

5. StringBuilder - 많은 문자열 결합 시  
   append() 메서드로 결합한 결과를 내부 메모리(버퍼)에 담아 두고 toString()으로 결과를 얻음

```kotlin
val sb = StringBuilder("Kotlin")
sb.append(" and ")
    .append("Android")
println(sb.toString())
```

### `+` 연산자가 느린 이유

String 인스턴스는 불변 객체(Immutable Object)

```kotlin
    val time = measureTime {
    var string = ""
    repeat(100000) {
        string += it.toString()
    }
}
println(time) // 3.18s

val time2 = measureTime {
    val sb = StringBuilder("")
    repeat(100000) {
        sb.append(it.toString())
    }
}
println(time2) // 5.89ms
```

## Accessor, Mutator

### Accessor (접근자) = Getter

Accessor는 객체의 private 변수 값을 읽기(조회) 위한 메서드입니다.

특징

- 일반적으로 get으로 시작하는 이름을 가짐 (예: getName(), getAge())
- 변수의 값을 **반환(return)** 하는 역할
- 변수의 타입과 동일한 반환 타입을 가짐
- 객체의 상태를 변경하지 않음 (읽기 전용)

```kotlin
val uppercased = river.uppercase()

```

### Mutator (변경자) = Setter

Mutator는 객체의 private 변수 값을 설정(변경) 하기 위한 메서드입니다.​

특징

- 일반적으로 set으로 시작하는 이름을 가짐 (예: setName(), setAge())
- 변수의 값을 설정하는 역할
- 반환 타입은 보통 void (또는 Unit)
- 객체의 상태를 변경함
- 유효성 검사를 추가할 수 있는 중요한 지점

### 코틀린은 Accessor와 Mutator를 자동으로 생성합니다.

```kotlin
class Person(
    val name: String,      // val: getter만 생성 (읽기 전용)
    var age: Int           // var: getter와 setter 모두 생성
)
```

### 요약

| 개념       | 다른 이름  | 역할      | 명명 규칙    | 반환 타입     |
|----------|--------|---------|----------|-----------|
| Accessor | Getter | 값 읽기    | get변수명() | 변수 타입     |
| Mutator  | Setter | 값 설정/변경 | set변수명() | void/Unit |
