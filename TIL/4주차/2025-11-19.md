# 데이터 소스

## 1️⃣ 데이터 소스란?

- 데이터 소스(Data Source)
    - 앱이 사용하는 **원천 데이터**
    - 비유하면 **요리의 재료**와 같다
- 앱은 보통 **외부에서 데이터를 가져와서 화면에 보여준다**

### 데이터 소스의 역할

- 외부 데이터 저장소와 통신
- Raw 데이터 수신 및 처리
- CRUD 작업 수행 (Create / Read / Update / Delete)

### DataSource의 종류 (예시)

- Text
- File
- JSON
- XML
- CSV
- RDBMS
- NoSQL

---

## 2️⃣ 데이터와 데이터 소스의 차이

- 데이터(Data)
    - 앱이 실제로 사용하는 **값 그 자체**
- 데이터 소스(Data Source)
    - **데이터를 나에게 주는 역할**
    - 예: 파일, 텍스트, JSON, DB 등에서 값을 읽어온다

---

## 3️⃣ 이름 짓기 관례

- 어떤 이름이 좋은지는 **상황에 따라 다르다**
    - 저장소 위치 기준: `LocalDataSource`, `RemoteDataSource`
    - 기술 스택 기준: `RoomDataSource`, `NetworkDataSource` 등
- 인터페이스 / 구현체 이름 관례
    - 인터페이스: `TodoDataSource`
    - 구현체: `TodoDataSourceImpl` 처럼 `Impl` 접미사를 붙인다

---

## 4️⃣ CSV와 JSON의 쓰임 차이

- CSV
    - 단순 테이블 구조에 적합
    - 메모리를 적게 사용하고 빠른 편
- JSON
    - 계층 구조, 복잡한 데이터 구조 표현에 유리
- 어떤 것을 쓸지는 **상황에 따라 달라진다**

---

## 5️⃣ 데이터 소스 예시

```kotlin
// 데이터 (값 그 자체)
data class Todo(
    val userId: Int,
    val id: Int,
    val title: String,
    val completed: Boolean
)

// 데이터 소스 (데이터를 가져오는 역할)
interface TodoDataSource {
    suspend fun getTodo(): Todo
    suspend fun getTodos(): List<Todo>
}

// 구현체 예시
class TodoDataSourceImpl(
    val file: File
) : TodoDataSource {
    override suspend fun getTodo(): Todo {
        return Json.decodeFromString(file.readText())
    }

    override suspend fun getTodos(): List<Todo> {
        return Json.decodeFromString(file.readText())
    }
}
```
